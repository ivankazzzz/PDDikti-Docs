Dokumentasi ini menjelaskan cara menggunakan endpoint `Profil Dosen` dan bagaimana struktur response yang akan Kamu terima. Dengan endpoint ini, Kamu dapat melihat detail informasi profil dosen berdasarkan `id_dosen`.

## Endpoint

Untuk mendapatkan detail data Profil Dosen, Kamu dapat menggunakan endpoint berikut:

```plaintext
GET /dosen/profile/<str:id_dosen>/
```

Pada endpoint ini, `id_dosen` adalah ID unik dari dosen yang ingin dicari profilnya.

## Contoh Endpoint

Untuk memberikan gambaran yang lebih jelas, berikut adalah contoh penggunaan endpoint:

```plaintext
GET /dosen/profile/IQo6xVoM7GH9q3uC1X3uxNKWyaVjUFZPwj1dFHbt4JRKXgzta8ztf1W6I7_HEos-8sIf8A==/
```

> Dalam contoh ini, API akan mencoba mengambil detail data profil dosen yang memiliki ID `IQo6xVoM7GH9q3uC1X3uxNKWyaVjUFZPwj1dFHbt4JRKXgzta8ztf1W6I7_HEos-8sIf8A==`.

    import requests

        try:

            response.raise_for_status()
            return response.json()
        except requests.exceptions.RequestException as e:
            print(f"Error: {e}")

    Kamu dapat mengubah `id_dosen` sesuai kebutuhan kamu.

    ```php:main.php showLineNumbers {15-16}
    <?php
    function fetch_api($api_url) {
        $ch = curl_init();
        curl_setopt($ch, CURLOPT_URL, $api_url);
        curl_setopt($ch, CURLOPT_RETURNTRANSFER, 1);
        $response = curl_exec($ch);
        if (curl_errno($ch)) {
            echo 'Error: ' . curl_error($ch);
            return null;
        }
        curl_close($ch);
        return json_decode($response, true);
    }

    $url = "https://api-pddikti.vercel.app/dosen/profile";
    $id_dosen = "IQo6xVoM7GH9q3uC1X3uxNKWyaVjUFZPwj1dFHbt4JRKXgzta8ztf1W6I7_HEos-8sIf8A==";
    $api_url = $url . '/' . urlencode($id_dosen) . '/';
    $data = fetch_api($api_url);

    if ($data) {
        echo '<pre>' . json_encode($data, JSON_PRETTY_PRINT) . '</pre>';
    } else {
        echo "No data fetched.";
    }
    ?>
    ```

    - **Mengirim Permintaan**: Fungsi `fetch_api` mengirim permintaan GET ke URL API dan mengembalikan data JSON.
    - **Menggabungkan URL**: `url` dan `id_dosen` digabungkan untuk membentuk URL lengkap.
    - **Mencetak Hasil**: Data yang diambil dicetak dalam format JSON yang rapi atau pesan kesalahan jika data tidak ditemukan.
